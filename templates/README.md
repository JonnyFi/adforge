# <your project>

Generated by [adforge](https://github.com/JonnyFi/adforge) — brief-to-live-ad pipeline.

## Quickstart

```bash
cp .env.example .env         # fill in keys
# then, in any agent CLI:
claude                       # or: codex, or: cursor
```

In the agent, say: **start adforge** (or `/adforge` in Claude Code).

The hub will walk you through what to do.

## Project layout

```
adforge.config.json     engine + adapter registry
brand.json              colors, fonts, voice
variants/               ad specs (one JSON per creative)
engines/
  static/               PIL composer — advertorial, stat-card, quote-card
  motion/               Remotion project — ops-console, product-mockup, walkthrough
adapters/
  meta/                 deploy / review / actions (Meta Graph API)
outputs/                rendered PNGs and MP4s
.adforge/state.json     Meta IDs, tracked by name (idempotent)
```

## Direct script usage (if you don't want the agent)

```bash
# static
python3 engines/static/compose.py variants/example.json 4x5 outputs/static/example_4x5.png

# motion
./engines/motion/render.sh variants/ops-console-example.json ops-console outputs/motion/ops.mp4

# deploy
python3 adapters/meta/deploy.py --dry-run adapters/meta/example-plan.json
python3 adapters/meta/deploy.py adapters/meta/example-plan.json

# review
python3 adapters/meta/review.py --days 7
```

## Adding engines or adapters

Drop new modules in `engines/` or `adapters/`, register them in `adforge.config.json`, and the agent skills will discover them.
